# 메뚜기 자동매매 v2 (KIS 기반, 신규 설계)

## 0. 프로젝트 정체성
- 한국투자증권(KIS) Open API 기반 한국 주식 자동매매 시스템
- 이전 v1 프로젝트의 코드, 구조, DB 스키마는 일절 재활용하지 않는다. 매번 새로 설계 검토한다.
- 모의투자 → 소액 실계좌 → 정식 운용 단계로 검증 후 전환
- 프로젝트 폴더: `C:\python\auto_trader_v2\`

## 1. 절대 원칙
1. 안정성 > 속도 > 기능 수. 충돌 시 항상 안정성 우선.
2. 모듈 분리: 전략 / 주문 / 데이터 / 리스크 / UI / 로그 / 설정. 책임을 섞지 않는다.
3. 브로커 추상화: `BrokerInterface`를 두고 KIS 구현체를 끼운다.
4. 모의/실계좌 명확 분리: 코드, 설정, UI에서 분리한다. 실계좌 주문은 별도 확인 게이트를 둔다.
5. 모든 주문은 중복 방지: 주문 키(종목+유형+시각) 기반 가드 + DB 영속화
6. 모든 상태는 영속화: 재시작해도 보유 포지션/주문 상태 복구 가능
7. 장애 = 정상 흐름의 일부: 로그인 실패, API 지연, 미체결, 주문 실패는 기본 시나리오로 처리
8. 하드코딩 금지: 모든 파라미터는 설정 파일/UI에서 조정 가능
9. 단계적 개발: 각 Phase는 독립적으로 동작하고 검증 가능해야 하며, 다음 Phase 전 검증 필수
10. 이전 v1 코드 패턴 답습 금지

## 2. 기술 스택
- 백엔드: Python 3.11+, FastAPI
- 프론트엔드: React (대시보드, 설정 UI)
- DB: SQLite (포지션, 주문, 체결, 로그, 설정 스냅샷, 1차 필터 풀, MACD 선계산)
- API: 한국투자증권 KIS Open API (REST + WebSocket)
- 알림: Windows 팝업 (`winotify`) → 추후 Telegram 확장
- OS: Windows

## 3. 모듈 구조
- `broker/`: `BrokerInterface` + KIS 구현체 (REST/WS 래퍼)
- `market/`: 시세, 일봉, 분봉 조회, 종목 마스터 관리
- `filter/`: 1차 필터 (제외종목 + 가격범위)
- `strategy/`: 매수 조건 판단 (타이밍 1/2), 실행과 분리
- `scanner/`: 1차 풀 기반 스케줄러
- `trader/`: 주문 실행, 미체결 관리, 재시도
- `position/`: 보유 포지션 관리, 손절/익절/MACD 매도 감시
- `risk/`: 일일 손실 한도, 동시보유 한도, 중복진입 쿨다운, 시간 가드, Kill Switch
- `notifier/`: 윈도우 팝업 (확장: 텔레그램)
- `storage/`: SQLite Repository 계층 (모듈은 SQL 직접 호출 금지)
- `logger/`: 카테고리별 로그 (`scan/order/fill/error/system/login`)
- `config/`: 설정 로드, 저장, 검증
- `api/`: FastAPI 엔드포인트
- `ui/`: React 대시보드, 설정 UI

## 4. 전체 실행 흐름
### 장 마감 후 (15:40~)
1. 당일 15분봉 마지막 캔들까지 데이터 수집
2. 매수 후보군(타이밍 1 수렴 확인 + 타이밍 2 상한가 신고가) MACD 선계산 후 저장

### 장 시작 전 (08:30)
1. 1차 필터 풀 갱신 (제외종목 + 가격범위)
2. 결과 SQLite 저장
3. 휴장일/반장일 체크 후 휴장이면 시작 거부

### 장중 (09:00~15:30)
1. 1차 풀 기반으로 매수 타이밍 1/2 실시간 감시
2. 매수 트리거 발동 시 리스크 가드 확인 후 매수
3. 보유 종목은 손절/익절 실시간 감시 + MACD는 15분봉 완성 시점마다 체크
4. 12:00 이후 신규 매수 중단 (매도는 계속)

## 6. 매수 전략
### 6.1 매수 타이밍 1 — 눌림목 돌파형
#### 조건 1: 일봉 추세
- 20MA(오늘) > 20MA(5거래일 전)
- 60MA(오늘) > 60MA(5거래일 전)

#### 조건 2: 강세일 존재 (최근 5거래일 내)
같은 날에 아래 둘 다 만족하는 일봉:
- `(종가 − 전일종가) / 전일종가 ≥ +15%`
- `종가 > 시가`
- `거래량 ≥ 20일 평균거래량 × 200%`

#### 조건 3: 수렴 (15분봉)
- 감시 구간: 강세일 다음 거래일부터 현재까지 (강세일 당일 제외)
- 15분봉 완성 캔들 기준:
- `종가 ≥ 20MA` AND `종가 ≥ 60MA`
- `max(종가, 20MA, 60MA) − min(종가, 20MA, 60MA) ≤ 종가 × 2%`
- 한 번이라도 만족 시 "수렴 확인" 기록 (장중 즉시 확정, 당일 재확인 없음)

#### 조건 4: 매수 트리거 (수렴 확인일 다음 거래일)
- 기준가 = 수렴 확인일의 일봉 고가
- 감시 시간: 09:00 ~ 12:00 (설정값)
- 실시간 체결가가 기준가 이상 되는 첫 순간 즉시 매수
- 12:00까지 미발동 시 당일 후보 폐기

#### 쿨다운
- 매수 신호 발동 또는 미발동 폐기 후 해당 강세일 기반 신호는 5일간 재진입 금지

### 6.2 매수 타이밍 2 — 상한가 신고가 재돌파형
#### 전일 조건
- 전일 종가 = 공식 상한가 가격 (정확히 일치, 종가 유지)
- 전일 종가가 최근 60거래일 이상 기준 최고가

#### 당일 매수 트리거 (실시간)
- 감시 시간: 09:00 ~ 12:00 (설정값)
- 기준시가 = 당일 시가
- 허용폭 = 설정값 (기본 0.3%)
- 순서대로 발생 시 매수:
1. 실시간 체결가 ≥ 기준시가 × (1 + 허용폭) → 상승 확인
2. 실시간 체결가 ≤ 기준시가 × (1 − 허용폭) → 하락 확인
3. 실시간 체결가 ≥ 기준시가 → 재돌파, 매수
- 12:00까지 미발동 시 당일 후보 폐기

### 6.3 우선순위
- 타이밍 1과 2 동시 해당 시 타이밍 1 우선
- 동일 종목 중복 진입 금지 (보유 중이거나 당일 체결 이력 있으면 스킵)

## 7. 매도 전략
### 7.1 매도 조건 (셋 중 가장 먼저 걸리는 것)
#### A. 손절 (실시간)
- 현재가 ≤ 매수가 × (1 − 손절%) → 즉시 시장가 매도
- 기본값 3%, 설정값으로 조정 가능

#### B. 익절 (실시간)
- 현재가 ≥ 매수가 × (1 + 익절%) → 즉시 시장가 매도
- 기본값 5%, 설정값으로 조정 가능

#### C. MACD 감소 (15분봉 완성 캔들 기준)
- 파라미터: `(12, 26, 9)` 기본
- 판정: `hist[t−2] > hist[t−1] > hist[t]` (2봉 연속 감소)
- 매 15분봉 완성 시점에 체크, 진행 중 봉 제외

### 7.2 매도 우선순위
- 손절/익절은 실시간, MACD는 15분봉 완성 시점만
- 동일 시점 충돌 시: 손절 > 익절 > MACD

### 7.3 MACD 데이터 선계산
- 장 마감 후 다음 거래일 매수 후보군에 대해 MACD 과거 값 미리 계산
- 대상: 타이밍 1 수렴 확인 완료 종목 + 타이밍 2 상한가 신고가 종목
- 매수 체결 시점부터 선계산 데이터에 이어 계산 시작

## 8. 설정 가능 항목
- 1차 필터: 가격 하한/상한, 거래대금 최소값
- 매수 타이밍 1: 강세일 상승률(기본 15%), 거래량 배수(기본 2배), 탐색 기간(기본 5일), 이평선 종류(20/60), MA 기울기 판정 거리(기본 5일), 수렴 허용치(기본 2%), 감시 시간(기본 09:00~12:00), 쿨다운(기본 5일)
- 매수 타이밍 2: 신고가 기간(기본 60일), 허용폭(기본 0.3%), 감시 시간(기본 09:00~12:00)
- 매도: 손절%(기본 3%), 익절%(기본 5%), MACD 파라미터(기본 12/26/9), 연속 감소 봉수(기본 2봉)
- 주문: 종목당 매수 금액, 동시 보유 종목 수, 일일 최대 주문 수, 일일 최대 손실
- 알림: on/off
- 모드: 모의/실계좌

## 9. 리스크 가드
- 중복 주문 방지 (메모리 캐시 + DB 키)
- 동일 종목 재진입 쿨다운 (타이밍 1 기준 5일)
- 미체결 주문 N초 후 자동 취소 (재시도 횟수 제한)
- 일일 최대 손실 도달 → 신규 매수 정지 (보유 종목 매도는 계속)
- 동시 보유 종목 수 초과 → 신규 매수 차단
- 매수 금지 시간대 (12:00 이후)
- Kill Switch: UI 긴급 정지 버튼 → 즉시 자동매매 중단
- 휴장일/반장일 자동 체크 후 시작 거부

- DB에도 핵심 이벤트 인덱싱

## 12. 단계별 개발 계획
- Phase 0: 프로젝트 골격 + KIS 인증(모의) + 토큰 자동 갱신
- Phase 1: `BrokerInterface` 추상화 + KIS 구현체 (잔고/시세/일봉/15분봉 조회)
- Phase 2: SQLite 스키마 + Repository 계층 + 설정 시스템
- Phase 3: 1차 필터 풀 갱신 (장 시작 전 자동)
- Phase 4: 매수 타이밍 1/2 조건 검사 (조회, 판정만. 주문 X) — 실제 후보 빈도 관찰
- Phase 5: 알림 + 수동 매수 버튼
- Phase 6: 자동 매수 + 중복 방지 + 미체결 처리
- Phase 7: 매도 로직 (손절/익절/MACD) + MACD 선계산
- Phase 8: 리스크 가드 (일일 손실, 동시 보유, Kill Switch)
- Phase 9: 대시보드 UI + 로그 뷰어
- Phase 10: 패키징 (다른 PC 설치 가능)
- Phase 11: 실계좌 전환 체크리스트

### 각 Phase 완료 기준
- 단위 동작 검증 완료
- 예외 케이스 테스트 완료 (실패, 지연, 중복 등)
- 다음 Phase 의존 인터페이스 안정화
